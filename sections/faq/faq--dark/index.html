Still stuck on something?

01 Is the library actually free, or is there a paid tier waiting?
All of it is free, MIT-licensed, forever. Every section is a single self-contained HTML file with no build step and no external JavaScript. Copy the markup, paste it into your project, ship it commercially — no attribution required, though a star never hurts.
02 Do I need Tailwind, React, or a bundler to use these?
No. Each file ships with a :root token block and plain component CSS in a single <style> tag. The only external dependency is a Google Fonts <link>, which you can swap for a system stack in one line. Drop it into vanilla HTML, Astro, Next, Rails — anything that renders markup.
03 How do I recolor a section to match my brand?
Edit the seven custom properties at the top of the file — --bg, --ink, --muted, --line, --accent plus the two font variables. Every color in the section references those tokens, so one accent change cascades through borders, markers, and hover states without touching the rest of the CSS.
04 Are the components accessible and keyboard-friendly?
This FAQ is built on native <details>/<summary> elements, so it toggles with Enter or Space, exposes correct expanded state to screen readers, and works with JavaScript disabled. Across the library we lean on semantic HTML first and add ARIA only where the platform can't carry it.
05 Will Claude Code understand a section if I paste it in raw?
Yes — that's the whole point. Files are small, flat, and heavily class-named (block__element convention), so an agent can read the structure, rename tokens, or splice two sections together without guessing. Paste the file and describe the change in plain English.
06 Can I mix this dark section with a light-themed page?
Absolutely. Because each section scopes its variables and background to its own root class, dropping a dark FAQ beneath a light hero just works — no bleed, no reset collisions. For a full light variant, override the five color tokens or grab the faq--light file from the same folder.
