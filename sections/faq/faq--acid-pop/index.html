01 Is this thing actually free, or is there a catch on line 40?
Fully free, no signup, no watermark, no phone-home script. Every section is a single self-contained index.html — copy it, paste it, ship it. Use it for client work, resell the finished page, whatever. We only ask you don't repackage the raw library and sell it as your own.
02 Do I need a build step, npm, or a framework to run these?
Zero build. No Tailwind config, no Vite, no node_modules. Open the file in a browser and it works. The only external dependency is a Google Fonts <link>, and you can swap that for a local font in about ten seconds if you're going offline-first.
03 How do I recolor a section to match my own brand?
Every file leads with a :root{} token block — --bg, --ink, --accent, and friends. Change those seven variables and the whole section repaints. Tell Claude Code "swap the tokens to my palette" and paste your hex codes; it'll rewrite the block cleanly without touching the markup.
04 Will these break on mobile or in older browsers?
Each section ships with a real @media(max-width:640px) block — not a squished afterthought. Layouts are grid and flexbox, so anything from the last four years of Chrome, Safari, Firefox, and Edge renders it correctly. The accordion here uses native <details>, so it even works with JavaScript disabled.
05 Can I combine multiple sections into one full page?
Yes — that's the whole point. Grab a hero, a features block, this FAQ, and a footer, then hand them to Claude Code with "merge these into one page and dedupe the :root tokens." Because they share the same token naming convention, they stitch together without a fight.
06 What's the catch with the fonts and licensing?
Everything ships on Google Fonts (Archivo here) under the SIL Open Font License — free for commercial use, no attribution required. The section code itself is MIT-style: keep it, fork it, sell the page you build with it. If a section ever references an icon or shape, it's an inline SVG or pure CSS, so there's nothing extra to license.
